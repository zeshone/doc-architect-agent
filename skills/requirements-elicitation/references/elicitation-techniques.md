# Elicitation Techniques Reference

Use this reference in Step 2 when selecting techniques and Step 3 when planning sessions. Match the technique to the information need, not to personal preference.

---

| Technique | When to Use | Participants | Output | Time |
|-----------|-------------|--------------|--------|------|
| **One-on-one Interview** | Deep domain knowledge, sensitive topics, executive stakeholders | 1 BA + 1 stakeholder | Interview notes, requirement statements | 45-60 min |
| **Facilitated Workshop** | Cross-functional alignment, conflicting viewpoints, complex processes | 1 BA + 5-12 stakeholders | Prioritized requirements, process maps | 2-4 hours |
| **Observation / Apprenticing** | Tacit knowledge, actual vs. documented processes, workarounds | 1 BA observing 1-3 users | Process notes, pain points, workaround list | 2-8 hours |
| **Document Analysis** | Existing policies, regulations, legacy system documentation | 1 BA | Extracted rules, constraints, data definitions | 1-4 hours |
| **Survey / Questionnaire** | Large user populations, quantitative validation, geographically distributed teams | 1 BA designs, many respondents | Statistical summary, preference rankings | 1-2 weeks |
| **Focus Group** | User experience feedback, feature prioritization, persona validation | 1 BA + 6-10 users | Grouped themes, priority rankings | 1-2 hours |
| **Prototyping** | UI/UX requirements, validating understanding, resolving ambiguity | 1 BA + 1-5 users | Annotated prototype, feedback log | 2-4 hours |
| **Brainstorming** | Early discovery, generating options, creative problem solving | 1 BA + 3-8 stakeholders | Idea list (unfiltered), affinity groups | 30-60 min |
| **Mind Mapping** | Exploring scope, connecting related concepts, structuring domain knowledge | 1 BA + 1-5 domain experts | Visual concept map | 30-60 min |
| **Interface Analysis** | System integration requirements, API contracts, data exchange | 1 BA + 1-2 technical leads | Interface specification, data mapping | 1-2 hours |
| **Business Rules Analysis** | Decision logic, validation rules, calculation formulas | 1 BA + domain expert | Decision tables, rule catalogue | 1-2 hours |
| **Competitive Analysis** | Market context, feature benchmarking, industry standards | 1 BA (desk research) | Comparison matrix, gap list | 4-8 hours |
| **User Story Workshop** | Agile contexts, collaborative requirement definition, sprint planning input | 1 BA + team (3-8) | User story backlog, acceptance criteria | 2-4 hours |
| **Context Diagram / Scope Model** | System boundary definition, identifying adjacent systems and actors | 1 BA + 1-3 architects | Context diagram, actor list | 1-2 hours |
| **Reverse Engineering** | Legacy system replacement, undocumented systems, migration projects | 1 BA + 1-2 developers | Feature inventory, business rule extraction | 4-16 hours |
| **Job Shadowing** | Understanding end-to-end workflows, user empathy, field operations | 1 BA shadowing 1 user | Day-in-the-life narrative, pain point inventory | 4-8 hours |

## Technique Selection Guide

**Start with these 3 for any project:**
1. **Interviews** - build individual understanding and trust
2. **Workshop** - align stakeholders and surface conflicts
3. **Document Analysis** - ground your work in existing constraints

**Add based on context:**
- Users can't articulate needs → **Observation / Job Shadowing**
- Large user base → **Survey** + **Focus Group**
- Complex UI → **Prototyping**
- System integration → **Interface Analysis**
- Legacy replacement → **Reverse Engineering**
- Agile team → **User Story Workshop**
